# 经验分享：AI 百分之百还原设计稿

> 核心思路：**不让 AI 凭空写 UI，而是让设计工具先出静态代码，再让 AI 将"死代码"转化为结构良好、自适应的生产级前端代码。**

---

## 为什么不直接让 AI 看截图写代码？

- AI 从截图还原 UI 看似简单，但**还原度极不稳定**——间距、字号、颜色、圆角等细节经常偏差
- 设计稿转代码工具（Figma Dev Mode / 蓝湖 / MasterGo）能**精确提取设计 Token**（颜色值、间距、字号），比 AI 看图猜值靠谱得多
- 让 AI 做它擅长的事（代码重构、逻辑编排），让工具做它擅长的事（像素级数值提取）

---

## 适用的设计工具

**任何支持"设计稿转代码"的工具都适用本流程**——无论是内置功能还是第三方插件。以下列举几个常见选项：

| 工具 | 转码方式 | 适用场景 |
|------|----------|----------|
| **Figma** | 内置 Dev Mode 导出 + 插件（如 Locofy、Anima） | 团队协作、国际化项目 |
| **蓝湖** | 内置标注 + 代码导出（HTML/CSS） | 国内团队常用，与 Sketch/PS 联动 |
| **MasterGo** | 内置设计转代码 + 插件生态 | 国内替代 Figma，转码能力较新 |
| **Sketch** | 通过插件（如 Sketch2Code）导出 | macOS 设计师常用 |
| **其他** | Zeplin、Locofy、Anima 等独立插件/平台均可 | 只要能导出 HTML/CSS 或框架代码即可 |

> 💡 很多设计工具本身不内置转码功能，但可以通过**安装第三方插件**实现（如 Figma 的 Locofy 插件、Anima 插件等）。选择工具时，优先看其插件生态是否支持你的技术栈。

> 不同工具的转码产物大同小异：**绝对定位 + 硬编码像素值 + 扁平 DOM 结构**。这些"死代码"不能直接用于生产，但**保留了精确的样式数值**，是 AI 改造的完美输入。关键不在于用哪个工具，而在于后续的 AI 转化流程。

## 给设计师的建议：制图方式决定转码质量

**设计师怎么画，直接决定转码工具能导出多干净的代码。** 同样的视觉效果，不同的实现方式会导致转码结果天差地别。

### 核心原则：用工具的原生属性，而非拼凑

| 视觉效果 | ❌ 拼凑式（转码后代码混乱） | ✅ 原生式（转码后代码干净） |
|----------|----------------------------|----------------------------|
| 圆角按钮 | 矩形 + 两个半圆拼接 → 导出 3 个 div | 矩形设置 border-radius → 导出 1 个 div + border-radius |
| 阴影 | 用半透明图层模拟阴影 → 导出多余图层 | 用工具内置 Drop Shadow → 导出 box-shadow |
| 渐变背景 | 多个纯色图层叠加 → 导出多个 div | 用渐变填充 → 导出 linear-gradient |
| 间距控制 | 用空白矩形撑开间距 → 导出多余空 div | 用 Auto Layout / 间距属性 → 导出 gap / padding |
| 图标 | 用基础图形拼凑图标 → 导出一堆 path | 合并为单个矢量图形 → 导出一个完整 SVG |

### 合理的图层结构

- **使用 Auto Layout（自动布局）**：这是转码质量的最大杠杆。Auto Layout 的结构会直接映射为 Flexbox，远比手动定位的图层导出效果好
- **合理分组**：视觉上属于同一组件的元素放在同一个 Group / Frame 中，转码工具才能识别组件边界
- **避免不必要的嵌套**：图层层级越深，导出的 DOM 层级越深，后续 AI 清理的工作量越大

> 💡 **设计师懂一点前端，转码效果会好很多。** 不需要会写代码，但理解"CSS 能原生实现哪些效果"（圆角、阴影、渐变、Flex 布局），就能在设计时自然选择工具的正确实现方式，让转码产物从源头就是干净的。

## 完整流程 SOP

### 第一步：设计稿转代码（工具侧）

1. 在设计工具中选择目标页面/组件
2. 使用内置的"转代码"或"开发模式"功能导出
3. 下载导出的代码文件（通常是 HTML + CSS，部分工具支持 React/Vue）
4. 将下载的代码文件**放入项目的临时目录**（如 `src/_design-draft/`）

**注意事项**：
- 导出时选择与项目技术栈匹配的格式（React / Vue / 纯 HTML）

### 第二步：AI 转化为生产级代码

将导出的静态代码喂给 AI，使用以下 Prompt 模板：

```
【任务】将设计稿导出的静态代码转化为生产级前端代码。

【输入文件】
- [粘贴或 @引用导出的代码文件路径]

【硬性要求】
1. **样式保持不变**：颜色、字号、间距、圆角、阴影等视觉属性必须与原始代码一致，禁止擅自"优化"
2. **消除绝对定位**：将 position: absolute 布局改为 Flexbox / Grid，保持视觉效果不变
3. **自适应布局**：将硬编码的固定尺寸改为自适应，根据项目实际方案处理
4. **语义化命名**：文件名、class 名、图片资源名等必须语义化，禁止保留工具生成的无意义命名
5. **合理拆分组件**：按职责拆分为独立的子组件，不要把整个页面塞进一个文件
6. **符合 Clean Code / KISS 原则**

```

> **核心原则**：工具负责精确提取样式数值，AI 负责代码结构重构，人工看效果确认。三者各司其职，才能实现百分之百还原。
